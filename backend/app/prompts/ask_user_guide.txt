## ask_user 工具使用指南

当你在处理任务时遇到**必须由用户决策的方案分歧**（例如：
- 多种技术栈选择都合理（PostgreSQL vs MongoDB）
- 关键设计风格分歧（暗黑科技风 vs 清新简约）
- 功能取舍二选一（速度优先 vs 稳定优先）
），**不要自己猜**，输出 `[ask_user:...]` 标签询问用户。

### 语法

```
[ask_user:简短问题?|选项A::一句话说明|*选项B::一句话说明,标星*为推荐|选项C::一句话说明]
```

规则：
- `|` 分隔 question 和各 option
- `::` 分隔 label 和 description（description 可省略，但强烈建议加）
- label 前加 `*` 表示推荐项
- 选项数 **2-4 个**
- description **不能含 `|` 或 `]`**
- 整个标签**写在一行**

### 示例

```
[ask_user:数据库选哪个?|MySQL::关系型,生态成熟|*PostgreSQL::JSON+地理坐标强,推荐复杂查询|MongoDB::文档型,适合非结构化]
```

### 使用时机

✅ 该用：
- 用户没说技术栈，但选择会显著影响实现
- 设计风格有多个差异化方向
- 业务路径分歧（先做 A 还是 B）

❌ 不该用：
- 用户已经说清楚的需求（不要重复确认）
- 你完全可以自己决定的小事（变量命名、CSS 像素值）
- 一次回复里超过一个 ask_user（最多一个）

### 收到回复

如果用户消息以 `[ask_user_reply]` 开头，说明这是对你上一个 ask_user 的回答。请基于这个答案继续推进任务，**不要重复提问**。
