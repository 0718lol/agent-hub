# Coze 极简聊天界面改造 — 设计方案

> 日期：2026-05-23  
> 目标：将 AgentHub 三栏布局改造为 Coze 风格的极简 IM 聊天界面

---

## 一、整体布局架构

### 三档断点布局

| 断点 | ≥1280px | 768-1279px | <768px |
|------|---------|------------|--------|
| 侧边栏 | 始终可见 200px | 隐藏，汉堡菜单浮层 | 隐藏，汉堡菜单浮层 |
| 聊天区 | 居中，最大 700px | 居中，最大 600px | 全宽，padding 16px |
| 右侧面板 | 按钮可见，滑出 380px | 按钮可见，滑出 320px | 完全隐藏（功能改内联卡片） |

### 布局结构（宽屏）

```
┌────────┬────────────────────────────┬─────┐
│侧边栏   │      聊天区（居中700px）     │ →□ │
│~200px  │      两侧大量留白            │面板  │
│        │  ┌──────────────────┐      │按钮  │
│ Agent  │  │   聊天顶栏        │      │     │
│ 列表   │  │   消息列表        │      │     │
│ + 新建 │  │   输入栏          │      │     │
│ ⚙ 设置 │  └──────────────────┘      │     │
└────────┴────────────────────────────┴─────┘
```

### 关键变更 vs 当前
1. 右侧 CanvasPanel 不再常驻，改为顶栏按钮触发的滑出面板（380px）
2. 聊天区从"左对齐弹性填充"改为"居中最大宽度 700px"
3. 侧边栏从 272px 三段式精简为 ~200px 单列表
4. 滑出面板承载：代码 Diff + Web 预览
5. DAG 协作图、任务看板、部署状态 → 聊天区内联可折叠卡片

## 二、侧边栏

### 结构

```
┌──────────────────┐
│  🔍 搜索会话...   │  ← 搜索框（可折叠为图标）
│  [+] 新建对话     │  ← 弹出 Agent 选择面板
│──────────────────│
│  📌 前端开发      │  ← 置顶项带图钉图标
│  💬 群聊：开发组  │
│  💬 Claude Code  │
│  ...             │
│──────────────────│
│  ⚙ 设置           │  ← 底部固定
└──────────────────┘
```

### 交互细节
- 每个对话项 = 头像(36px, 圆角 8px) + 名称 + 时间戳（右侧辅助文字）
- 置顶项自然排在列表顶部，图钉图标使用 Lucide `Pin` 线性图标
- hover 对话项出现 `…` 菜单（Lucide `MoreHorizontal`）→ 弹出操作菜单（置顶/取消置顶、归档、删除）
- 归档对话不显示在列表中，仅通过顶部搜索框可找回
- 未读消息：名称加粗 + 左侧蓝色圆点（6px）
- 支持拖拽排序调整会话顺序
- 设置按钮固定在侧边栏底部，使用 Lucide `Settings` 图标

### Agent 选择面板（点击"+"弹出）
- 浮层 Modal，居中显示
- 网格排列内置 Agent（带头像、名称、一行角色描述）
- 点击 Agent → 立即创建单聊对话并切换
- 无"自定义 Agent"入口（保持纯 Coze 风格直接选择）

## 三、聊天顶栏

### 单聊模式

```
┌─────────────────────────────────────────────┐
│  🤖 前端工程师          Claude 4.5 ● 在线    │
│      前端开发专家，擅长 React/TypeScript      │
└─────────────────────────────────────────────┘
```

- 左侧：Agent 头像(36px, 圆角 8px) + 名称(18px/600) + 角色描述(12px/400/次要文字)
- 右侧：模型名称标签 + 在线状态指示灯(8px 圆点) + 面板切换按钮

### 群聊模式

```
┌─────────────────────────────────────────────┐
│  👥 开发团队  [头像叠放]  5人 · 2人在线       │
└─────────────────────────────────────────────┘
```

- 左侧：群聊名称 + 成员头像 2×2 网格叠放（微信风格，36px 总尺寸，小格 17px）
- hover 头像区域 → 弹出完整成员列表浮层（头像+名称+角色）
- 右侧：当前活跃 Agent 指示灯（脉冲动画）+ 在线人数 + 面板按钮

### 通用元素
- 最右侧固定：面板切换按钮，使用 Lucide `PanelRight`/`Sidebar` 图标
- 面板按钮：默认/点击打开状态用 `PanelRightOpen` / `PanelRightClose`

## 四、消息气泡

### Coze 极简扁平风格

- Agent 消息：左对齐，无背景色，纯文本，小头像在前
- 用户消息：右对齐，浅灰底（`#F2F3F5`），小圆角
- **气泡宽度必须跟随文字内容自适应**，禁止短文字配超宽气泡
- **高度跟随文字行数自适应**，禁止固定容器高度
- **同类组件内边距/行高/圆角全界面统一**

### 气泡规范

| 属性 | Agent 消息 | 用户消息 |
|------|-----------|---------|
| 背景 | 无 | `#F2F3F5`（深色 `#262627`） |
| 圆角 | N/A | 12px |
| 内边距 | 0 | 10px 14px |
| 最大宽度 | 85% 容器 | 75% 容器 |
| 头像 | 32px, 圆角 6px | 无 |

### 消息类型

| 类型 | 表现 |
|------|------|
| 普通文本 | Agent 无背景左对齐，用户浅灰底右对齐 |
| 代码块 | 浅灰背景卡片(`#F7F8FA`)，语言标签 + 复制按钮 |
| Diff 卡片 | 内联展示，绿色+行 / 红色-行，`[应用此 Diff]` 按钮 |
| 部署状态卡片 | 终端风格小卡片，日志最后 3 行 + 状态图标 + 展开 |
| 图片/文件 | 缩略图卡片，点击放大/下载 |
| 网页预览卡片 | OG 信息卡片（标题+描述+链接），点击右侧面板打开 |

### 消息操作（hover 出现）
- 回复（引用原消息）、复制（代码块）、重新生成、Pin 消息
- 所有操作按钮使用 Lucide 线性图标：`Reply`/`Copy`/`RefreshCw`/`Pin`

### 内联卡片（DAG / 任务看板 / 部署）

```
┌────────────────────────────────┐
│  [DAG图标] 协作图    [展开 →] │  ← 可折叠卡片
│  ┌──────────────────────────┐ │
│  │  PM → Designer → FE     │ │  ← 缩略版 DAG
│  │        ↘ BE → Tester    │ │
│  └──────────────────────────┘ │
│  3/5 任务完成 · PM 工作中      │
└────────────────────────────────┘
```

点击"展开" → 在聊天区内展开完整内容或跳转右侧面板。

## 五、输入区域

### 单聊模式

```
┌─────────────────────────────────────────────┐
│  ┌─────────────────────────────────┐ [发送] │
│  │ 输入消息...               [展开]│   →    │
│  └─────────────────────────────────┘        │
└─────────────────────────────────────────────┘
```

- 默认单行，高度随内容自动增长（最大 4 行）
- 发送按钮：Lucide `Send`/`ArrowUp` 图标，消息为空时置灰
- 展开按钮：Lucide `Maximize2` 图标，点击弹出大编辑框
- Enter 发送，Shift+Enter 换行

### 群聊模式

```
┌─────────────────────────────────────────────┐
│  ┌────────┬──────────────────────┐ [发送]   │
│  │   @    │ @前端 写一个按钮组件   │   →      │
│  └────────┴──────────────────────┘          │
└─────────────────────────────────────────────┘
```

- 左侧 @ 按钮：点击弹出 Agent 选择器（可多选）
- 选中 Agent 显示为蓝色 `@Agent名称` 标签（背景 `#E8F3FF`），可点击 × 删除
- 同时支持直接输入 `@` 触发选择器
- 发送按钮同上

### 通用规则
- 输入框边框：默认 `#E5E6EB`，聚焦 `#165DFF`
- placeholder：单聊"输入消息..."，群聊"@ 指定 Agent 或输入消息..."
- 所有图标使用 Lucide 线性图标

## 六、右侧滑出面板

### 触发
- 聊天顶栏右侧 ☰ 面板按钮，点击滑出
- 面板宽度 380px（768-1279px 为 320px），从右侧滑入，300ms 过渡
- 点击面板外区域或再次点击按钮 → 关闭
- 快捷键 `Ctrl+B` 切换
- **面板常驻选项**：开启后面板不随点击外部关闭，仅按钮或快捷键关闭

### 标签页（2 个）

**代码标签页：**
- 文件横向标签切换（激活态：文字高亮 + 底部 2px 下划线，无背景填充）
- Diff 高亮（+绿 / -红），行号显示
- 每个文件卡片右上角复制按钮（Lucide `Copy`）
- 底部浮动主操作按钮"应用所有 Diff"（主色调蓝色）

**预览标签页：**
- 顶部 URL 栏 + 刷新按钮（Lucide `RefreshCw`）
- iframe 沙箱预览，填充剩余空间

### 设计规范
- 所有操作按钮使用 Lucide 线性轮廓图标
- 标签页激活态仅文字高亮 + 下划线，无背景填充
- 卡片内边距与全界面排版规范对齐

## 九、响应式断点

### 三档断点

| 区域 | ≥1280px | 768-1279px | <768px |
|------|---------|-----------|--------|
| 侧边栏 | 始终可见 200px | 隐藏，汉堡菜单浮层 overlay | 隐藏，汉堡菜单浮层 |
| 聊天区 | 居中，最大 700px | 居中，最大 600px | 全宽，padding 16px |
| 右侧面板 | 按钮可见，滑出 380px | 按钮可见，滑出 320px | 完全隐藏 |
| 消息气泡最大宽 | ~85% / 75% | ~90% / 80% | ~95% / 85% |

### 气泡自适应规则
- 气泡宽度 = `min(max-content, 百分比上限)`，短文字不自适应拉宽
- 各断点气泡占比配合容器自适应，禁止留白空洞、文字撑不满容器

### 移动端适配要点
- 汉堡菜单：左上角 44px 触摸区域，Lucide `Menu` 图标
- 新建对话按钮移至聊天顶栏右侧
- 搜索框移至汉堡菜单浮层顶部
- 右侧面板功能（代码/预览）改为聊天区内联卡片
- 可点击区域最小 44px（触摸友好）

## 十、核心交互流程

### 新建单聊
```
侧边栏 [+] → Agent选择面板弹出 → 点击Agent → 进入空白对话 → 开始对话
```

### 群聊 @ 指派
```
群聊输入框 @按钮 / 输入@键 → Agent选择器浮层 → 勾选1-N个Agent →
蓝色@标签插入输入框 → 输入任务文本 → 发送 → Agent依次回复
```

### Pin 消息
```
hover 消息 → …菜单 → 固定（Lucide Pin图标） → 
消息顶部出现 Pin 标识（Lucide Pin 线性图标）→ 侧边栏/顶栏可查看已固定列表
```

### 重新生成
```
hover Agent消息 → 重新生成（Lucide RefreshCw图标）→ 
消息进入加载态 → 旧消息折叠（可展开）→ 新消息流式输出替换
```

### 拖拽排序
```
侧边栏会话项 → 长按/拖拽手柄 → 拖拽到目标位置 → 释放 → 顺序持久化
```

## 十一、技术实现要点

### 文件范围
- **新文件**：Coze 浅色主题 CSS、深色主题 CSS、滑出面板组件、Agent 选择器组件
- **重写文件**：`App.jsx`、`Sidebar.jsx`、`ChatPanel.jsx`、`InputBar.jsx`、`MessageBubble.jsx`、`global.css`
- **删除文件**：`theme-tech-dark.css`、`theme-vibrant.css`、`CanvasPanel.jsx` 及相关 Canvas 子组件（DAG/TaskBoard/DeployPanel 逻辑移入内联卡片，其余废弃）
- **修改文件**：`stores/canvasStore.js`（精简状态）、`themeStore.js`（适配新主题键值）

### 风险点
1. WebSocket 消息类型映射不受影响（后端不变）
2. CSS 变量命名保持与现有 token 兼容，减少 store 和组件改动
3. `themeStore` 的 localStorage key 不变（`agent-hub-theme`），但值从 `tech-dark`/`vibrant` 改为 `light`/`dark`
4. 拖拽排序需引入轻量库或原生 HTML5 Drag API
