# AgentHub 项目状态与需求分析

## 一、比赛基本信息

| 项目 | 内容 |
|------|------|
| 项目名 | AgentHub - 多Agent协作平台 |
| 比赛 | AI全栈挑战赛（字节跳动 Coze 部门主办） |
| 启动时间 | 2025年5月20日 |
| 提交截止 | 2025年6月10日 |

## 二、评分维度（按权重排序）

| 维度 | 权重 | 评判要点 |
|------|------|----------|
| AI协作能力 | **30%** | 沉淀 Prompt/Skill/Rules 协作规范文档（最重要！） |
| 功能完整度 | 25% | IM核心体验流畅、多Agent调度跑通 |
| 生成效果质量 | 20% | 聊天UI体验、产物预览效果 |
| 代码理解度 | 15% | 答辩时能解释架构选型和核心逻辑 |
| 创新与产品感 | 10% | 超预期功能点或体验优化 |

## 三、最终交付物（必须提交）

2. 技术文档
3. 可运行Demo

## 四、课题核心功能要求

### P0 必做功能

| 功能 | 说明 | 当前状态 |
|------|------|----------|
| 对话列表 | 左侧会话列表，支持新建/置顶/归档/搜索 | ✅ 基本完成 |
| 单聊模式 | 1v1与单个Agent对话 | ✅ 已完成 |
| 群聊模式 | 一个对话中包含多个Agent，通过@指定或由Orchestrator自动分派 | ✅ 已完成 |
| 主Agent协调器 | 自动理解用户意图，拆解任务并分派给子Agent | ✅ 已完成 |
| 多Agent接入 | 至少接入2个主流Agent平台（Claude Code + OpenCode） | ✅ 已完成 |
| 用户自建Agent | 对话式创建，设定System Prompt + 工具集 | ✅ 已完成 |
| 产物预览 | Agent回复中内联网页预览卡片（iframe） | ✅ 已完成 |
| 代码展示 | 代码块语法高亮 + 一键复制 | ✅ 已完成 |
| 上下文连续 | 聊天历史自动作为上下文传递给Agent | ✅ 已完成 |
| 消息操作 | 回复、引用、重新生成、复制代码 | ⚠️ 部分完成 |
| 流式输出 | Agent回复逐字显示 | ✅ 已完成 |
| 思考过程展示 | DeepSeek风格思考展示 | ✅ 已完成 |

## 六、技术栈

## 八、团队讨论要点

1. 先完成基础功能，再优化，展示效果决定能否进入答辩
2. 借鉴GitHub开源项目可以，但需有自己的思考和完整闭环
3. 移动端可通过网页端适配手机访问，不必做独立APP
4. 语音输入是好的创新点（简单但实用）
5. Best-of-N多候选方案已在项目中实现
6. Harness/Skill评价体系需自行设计并写入技术文档
7. 答辩时优先选择作品呈现好的团队
